**CORS allowlist — evil origin:**
```bash
curl -sI -H "Origin: https://evil.com" https://boss.xkmu.de/api/v1/health
```
Expected: NO Access-Control-Allow-Origin: https://evil.com in response (evil.com is not in allowlist).

**CORS allowlist — allowed origin:**
```bash
curl -sI -H "Origin: https://boss.xkmu.de" https://boss.xkmu.de/api/v1/health
```
Expected: Access-Control-Allow-Origin: https://boss.xkmu.de present.

**CORS preflight:**
```bash
curl -sI -X OPTIONS -H "Origin: https://boss.xkmu.de" -H "Access-Control-Request-Method: POST" https://boss.xkmu.de/api/v1/leads
```
Expected: 204 response with Access-Control-Allow-Methods, Access-Control-Allow-Headers.

**CVE-2025-29927 bypass test:**
```bash
curl -sI -H "x-middleware-subrequest: 1" https://boss.xkmu.de/api/v1/leads
```
Expected: 401 Unauthorized (from withPermission() in route handler — proxy.ts stripped the header but route still checks auth).

**Static asset speed (no proxy overhead):**
```bash
curl -sI "https://boss.xkmu.de/_next/static/chunks/main.js" 2>/dev/null | head -5
```
Expected: 200 or 304, no abnormal latency from proxy.

If Portainer has not redeployed yet, wait and retry. If any check fails, read the server response carefully and identify whether the issue is in next.config.ts (headers) or proxy.ts (CORS/CVE). Fix the relevant file, rebuild, and redeploy.
