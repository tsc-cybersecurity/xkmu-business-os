<objective>
Replace all 8 N+1 query patterns with batch operations.

Purpose: Each sequential `await` in a DB loop adds a full round-trip per item. With 50 audit answers, 20 sort-order updates, or 10 topics, this multiplies latency by N. Batch operations collapse N round-trips to 1.

Output: 8 modified files, zero `await` in DB loops, build passes.

IMPORTANT SCHEMA FACT: `dinAnswers` table has NO unique constraint on `(sessionId, requirementId)` — only non-unique indexes. Therefore, `INSERT ON CONFLICT DO UPDATE` will NOT work for din-audit. Use `Promise.all()` instead (parallelizes, does not reduce query count, but is safe and correct).
</objective>

Pattern 2 — CASE WHEN batch UPDATE (for reorder):
```typescript
import { sql, inArray, and, eq } from 'drizzle-orm'

async reorder(pageId: string, blockIds: string[]): Promise<boolean> {
  if (blockIds.length === 0) return true
  const caseExpr = sql`CASE id ${sql.join(
    blockIds.map((id, i) => sql`WHEN ${id}::uuid THEN ${i}`),
    sql` `
  )} ELSE sort_order END`
  await db.update(cmsBlocks)
    .set({ sortOrder: caseExpr, updatedAt: new Date() })
    .where(and(eq(cmsBlocks.pageId, pageId), inArray(cmsBlocks.id, blockIds)))
  return true
}
```
Note: Use `::uuid` cast to avoid PostgreSQL "operator does not exist: uuid = text" error.
