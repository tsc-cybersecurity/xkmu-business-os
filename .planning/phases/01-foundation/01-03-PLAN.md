<interfaces>
<!-- Docker compose variable substitution syntax -->

Docker compose supports two forms:
- `${VAR:-default}` — use default if VAR is unset or empty. INSECURE for secrets.
- `${VAR:?error message}` — exit with error if VAR is unset or empty. SECURE for required secrets.
- `${VAR:-}` — use empty string if unset. OK for optional variables.

Running `docker compose -f docker-compose.local.yml config` with a missing required var:
prints "invalid interpolation format: ... variable SUPABASE_DB_PASSWORD is required but not set"
and exits with code 1.

Current lines in docker-compose.local.yml that need changing (4 lines):

Also needs :? syntax (currently uses :-):
