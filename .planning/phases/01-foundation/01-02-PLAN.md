**Change 2 — Rewrite parseInsertStatements() to return columns + values instead of raw SQL strings:**

The function must parse an INSERT statement like:
```sql
INSERT INTO companies (id, name, tenant_id) VALUES ('uuid', 'Acme GmbH', 'tenant-uuid');
```

And return:
```typescript
{ table: 'companies', columns: ['id', 'name', 'tenant_id'], values: ['uuid', 'Acme GmbH', 'tenant-uuid'] }
```

Parsing approach:
1. Use a regex to extract the column list: `/\(([^)]+)\)\s+VALUES\s+\(/i` — captures `id, name, tenant_id`
2. Use a function `parseValuesList(valueStr: string): unknown[]` that handles:
   - `NULL` → `null`
   - `TRUE` → `true`, `FALSE` → `false`
   - `'some string'` → `'some string'` (strip outer quotes, unescape `''` → `'`)
   - `'{"key":"val"}'::jsonb` → parse with `JSON.parse()` (strip `::jsonb`)
   - `'2026-01-01T00:00:00.000Z'` (ISO date string) → `new Date('2026-01-01T00:00:00.000Z')` (check if string matches `/^\d{4}-\d{2}-\d{2}T/`)
   - bare numbers → `Number(token)`
3. The VALUES portion starts after `VALUES (` and ends before the final `)` (before the optional `ON CONFLICT` or `;`). Parse the comma-separated tokens carefully — string values may contain commas, so parse character by character tracking single-quote nesting.
4. Reject multi-row INSERT (multiple `VALUES (...)` clauses) by checking if the statement contains `), (` after the first VALUES tuple — log a warning and skip.

Full rewritten function signature:
```typescript
function parseValuesList(valuesStr: string): unknown[]
function parseInsertStatements(sqlContent: string): ParsedInsert[]
```
